Workflow Runtime View

This page explains the live automation path. The editable system workflow rules are now in Settings, like v18.

Continuous production loop:
1. Conveyor + Vision detects only RED / GREEN / BLUE cubes and fills the matching box.
2. Conveyor sends box filled event to the coordinator.
3. Coordinator reads the Workflow Rule for that color.
4. If another job is active, the filled box waits in the queue.
5. Coordinator starts the matching arm recipe: GREEN_BOX_A1, RED_BOX, or BLUE_BOX.
6. Arm moves the filled box from the conveyor onto the AMR.
7. Operator manually drives the AMR, drops the box, and returns the AMR.
8. Operator presses Finished Job in the AMR tab.
9. Coordinator resets that box and automatically starts the next queued filled box.

Workflow Rules

Map each filled color box to its SCARA arm recipe. AMR delivery is manual, and Finished Job releases the next queued box.

Example system flow:
RED_BOX_FLOW
  When conveyor sends: BoxRed_filled
  Wait before Arm: 0 ms
  Run arm recipe: RED_BOX
  Arm loaded event: BoxRed_loaded_on_amr
  Operator manually delivers and returns the AMR
  Operator presses: Finished Job
  Next queued filled box starts automatically.
